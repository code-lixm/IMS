# Interview Screening - PDF 智能评分助手

## Skill 简介

`interview-screening` 是一个面向招聘与文档评估的评分 skill：优先使用多模态模型直接理解 PDF 内容，结合**抱抱系统六维度评分（初筛阶段（S0）前置）**、红线风险识别与证据置信度，输出可解释、可复核、可批量处理的评估报告。

## 一句话简介（可直接对外使用）

在保留效率的同时提升公平性：`interview-screening` 能根据 PDF 与岗位标准，生成带证据等级和风险提示的结构化评分结论。

## 核心能力

- **多模态优先解析**: 直接读取 PDF（文本 + 版式），减少纯文本抽取误差
- **岗位化评分**: 支持技术岗、产品岗、运营岗三套子模板
- **平衡规则**: 使用 6 大维度，避免重复计分与单一学历偏置
- **风险防线**: 红线风险单独判定，不与普通分数混算
- **可解释输出**: 每个维度输出得分原因、证据等级、待核验项

## 适用场景

- 简历初筛与候选人排序
- 校招/社招申请材料评分
- 学术或项目文档质量评估
- 多份 PDF 批量打分与横向比较

## 工作流程

### Step 1: 收集评分输入

**必须输入**:

1. **PDF 文件路径**
2. **岗位轨道**: `tech` / `product` / `ops`
3. **评分模板**: 默认使用平衡版模板，可切换子模板
4. **输出格式**: 默认为 Markdown 表格

**缺失信息时，使用以下询问模板**:

```text
请补充以下信息：
1) 需要评分的 PDF 文件路径
2) 岗位轨道（tech / product / ops）
3) 是否使用默认模板（默认是）
4) 输出格式（默认 Markdown）
```

### Step 2: 读取 PDF（多模态优先）

**首选方法**:

```text
使用 read 工具直接读取 PDF
```

**备选方法（仅在无法直接读取 PDF 时，强制）**:

```text
调用 custom tool: interview_scanPdf(profile="screening", pdfPath="...")
```

> 禁止在 skill 流程中直接执行 `python3 .../parse_pdf.py` 命令，必须通过 `.opencode/tools/interview.js` 暴露的 `interview_scanPdf` 调用。

### Step 3: 执行评分

#### 3.0 抱抱系统六维度评分（前置，强制）

在任何筛选结论（推荐/储备/暂不推荐）生成前，必须先完成“抱抱系统六维度评分”并产出完整维度明细。

- 不允许跳过六维度直接给结论。
- 不允许只给总分不写维度证据。
- 六维度缺失任一项时，必须 fail-fast 并返回补充清单。

#### 3.1 红线风险先判定

先检查红线项（如法律合规、履历真实性重大疑点）。

- 命中红线：结论直接 `暂不推荐`
- 仍需给出完整评分过程，便于复核

#### 3.2 六维度评分（总分 100）

| 维度 | 满分 | 评分说明 |
|------|------|----------|
| 教育与资质 | 15 | 学历/专业/证书与岗位相关性 |
| 经历匹配度 | 20 | 行业与职责连续性、岗位贴合度 |
| 项目产出与业务价值 | 25 | 结果量化、贡献边界、业务影响 |
| 岗位核心能力 | 20~25 | 按岗位轨道评估核心能力 |
| 协作与领导 | 7~12 | 跨团队协同、推动与管理能力 |
| 稳定性与合规风险 | 8~10 | 履历一致性、稳定性、基础合规 |

> 不同岗位子模板会对部分维度分值做差异化配置。
> 本表即“抱抱系统六维度评分”在本项目的标准落地版本。

#### 3.3 证据置信度修正

每项维度都要标注 `evidence_level`：

- `high` = 1.00
- `medium` = 0.85
- `low` = 0.65

维度最终得分 = 维度原始得分 × 证据置信度系数。

#### 3.4 未知信息策略

缺失信息按 `unknown_not_zero` 处理：

- 不直接记 0 分
- 按该维度基准比例（默认 40%）计入
- 必须进入“待核验项”列表

#### 3.5 总分计算

```text
基础分 = 六维得分加总
最终得分 = 基础分 + 偏好加分 + 风险扣分
```

### Step 4: 输出评分报告

**报告结构（必含）**:

```markdown
### 一、分析结论

| 项目 | 评估结果 |
|------|----------|
| 最终得分 | XX/100 |
| 决策结论 | 推荐面试 / 建议储备 / 暂不推荐 |
| 红线命中 | 是/否 |
| 总体说明 | 一句话结论 |

### 二、红线风险核查

| 风险项 | 是否命中 | 证据 |
|--------|----------|------|

### 三、维度评分明细

| 维度 | 满分 | 原始得分 | evidence_level | 修正后得分 | 得分原因 |
|------|------|----------|----------------|------------|----------|

### 四、加减分项

| 类型 | 项目 | 分值 | 说明 |
|------|------|------|------|

### 五、待核验项

| 维度 | 缺失信息 | 对结论影响 |
|------|----------|------------|

### 六、面试建议考察维度

| 评估维度 | 建议提问 | 关注风险 |
|----------|----------|----------|
```

## 决策阈值（统一规则）

- **推荐面试**: `score >= 80` 且未命中红线风险
- **建议储备**: `65 <= score < 80`，或存在不超过 1 项关键待核验
- **暂不推荐**: `score < 65`，或命中任一红线风险

## 批量评分建议

1. 固定同一岗位轨道与模板
2. 对每份 PDF 输出完整维度明细与待核验项
3. 最后生成横向汇总表（分数、结论、核心风险）

## 注意事项

1. **多模态优先**: 优先 `read` 直接读取 PDF
2. **公平性优先**: 学历等背景项默认不一票否决（红线风险除外）
3. **未知不等于零分**: 缺失信息进入待核验，而非直接淘汰
4. **人工复核**: AI 评分用于初筛，最终结论建议人工确认
5. **隐私合规**: 处理简历与敏感文档时遵循最小暴露原则

## 与主流程契约（新增）

- 本 Skill 只负责返回筛选评分内容。
- 阶段主文档命名由 `interview-orchestrator + interview-records` 统一处理。
- 初筛阶段（S0）固定主文档名：`00_筛选报告.md`（见 `.opencode/agents/interview-manager.md`）。
- 初筛阶段（S0）必须先完成“抱抱系统六维度评分”再输出筛选结论。
