# 面试题目模板库

> 职级口径统一说明（强制）：
> - 本模板库仅适用于 `P5~P8`，并按 `-/标准/+` 三档制细分。
> - 禁止使用 `P9+`、`Senior`、`P6.5` 等越界或别名职级。
> - 具体推荐规则以 `interview-assessment` 中“职位体系（P5-P8，三档制）”为唯一标准。

## 第1轮：技术专家面试

> 调整重点：保留项目深挖、基础原理、设计实现三类题型，但把真实项目细节、方案取舍、复杂问题排查、性能稳定性与复盘意识放到主轴。

### P5-P6 级别（初级-中级工程师）

#### 题目类型1：项目技术点深挖
**场景引入**：从简历中的项目经验切入
**主问题**：请介绍一下你在[项目名]中负责的[技术模块]

**追问层级**：
- L1（事实确认）：这个模块的具体功能是什么？你在其中承担了什么角色？
- L2（原理深挖）：为什么选择[技术方案]？有没有考虑过其他方案？
- L3（发散验证）：如果让你重新设计，你会做哪些改进？

**参考答案要点**：
1. 能清晰描述项目背景和个人职责
2. 理解所选技术方案的核心原理
3. 能够对比不同方案的优缺点
4. 具备技术反思和优化意识

**评分标准**：
- 优秀：回答全面，主动深入，有独立见解
- 良好：回答完整，能回应追问，有一定深度
- 待观察：回答表面，需要引导，缺乏深度思考

#### 题目类型2：基础知识考察
**场景引入**：从简历中的技术栈切入
**主问题**：看你使用过[技术/框架]，能说说它的核心原理吗？

**追问层级**：
- L1（基础概念）：它的核心组件/机制是什么？
- L2（实现原理）：它是如何工作的？有哪些关键设计？
- L3（应用理解）：什么场景适合使用？有什么坑需要注意？

**参考答案要点**：
1. 掌握核心概念和术语
2. 理解底层实现机制
3. 了解适用场景和限制
4. 有实际踩坑经验

**评分标准**：
- 优秀：原理清晰，能画图说明，有深度理解
- 良好：概念正确，能举例说明，有实践经验
- 待观察：概念模糊，死记硬背，缺乏实践

#### 题目类型3：编码/设计能力
**场景引入**：从简历中的技术难点切入
**主问题**：假设[相关场景]，你会如何设计实现？

**追问层级**：
- L1（方案设计）：你的整体思路是什么？核心模块有哪些？
- L2（细节考量）：[具体场景]你会怎么处理？
- L3（优化思考）：如何保证[性能/扩展性/稳定性]？

**参考答案要点**：
1. 设计思路清晰，模块划分合理
2. 能考虑边界情况和异常处理
3. 关注性能、可维护性等质量属性
4. 有备选方案和权衡思考

**评分标准**：
- 优秀：设计完整，主动考虑边界，有优化意识
- 良好：方案可行，能回应追问，基本完整
- 待观察：思路混乱，遗漏重要考虑点

### P7-P8 级别（高级工程师/专家）

#### 题目类型1：系统架构设计
**场景引入**：从简历中的大规模项目切入
**主问题**：请介绍一下[项目]的整体架构设计

**追问层级**：
- L1（架构概览）：系统有哪些核心模块？它们之间如何协作？
- L2（设计决策）：为什么选择[架构方案]？解决了什么问题？
- L3（演进思考）：架构经历过哪些迭代？未来如何演进？

**参考答案要点**：
1. 能清晰描述系统全貌和核心链路
2. 理解每个架构决策的trade-off
3. 有架构演进和治理经验
4. 能识别当前架构的局限性

**评分标准**：
- 优秀：架构清晰，决策有理有据，有演进思考
- 良好：描述完整，能解释关键决策，有经验总结
- 待观察：描述混乱，决策缺乏依据，无演进意识

#### 题目类型2：复杂问题解决
**场景引入**：从简历中的技术挑战切入
**主问题**：当时遇到的最大技术挑战是什么？如何解决的？

**追问层级**：
- L1（问题定义）：具体是什么问题？影响范围有多大？
- L2（解决过程）：你尝试了哪些方案？如何决策的？
- L3（沉淀复盘）：这个经验对你后续工作有什么影响？

**参考答案要点**：
1. 能准确定义问题和量化影响
2. 展示系统的排查和解决思路
3. 体现技术决策能力
4. 有方法论沉淀和复用意识

**评分标准**：
- 优秀：问题清晰，解决过程系统，有深度复盘
- 良好：能描述问题和解决过程，有一定思考
- 待观察：问题描述模糊，解决过程随意，无复盘

#### 题目类型3：技术选型与权衡
**场景引入**：假设一个业务场景
**主问题**：如果让你设计一个[业务系统]，你会怎么选型？

**追问层级**：
- L1（技术选型）：存储/缓存/消息队列你会选什么？为什么？
- L2（权衡分析）：这些选型有什么优缺点？适用什么场景？
- L3（成本考量**：性能、成本、维护性如何平衡？

**参考答案要点**：
1. 选型有理有据，考虑业务特点
2. 能分析不同方案的trade-off
3. 关注成本、维护性等综合因素
4. 有实际踩坑经验支撑

**评分标准**：
- 优秀：选型全面，分析深入，有成本意识
- 良好：选型合理，能解释原因，有实践支撑
- 待观察：选型随意，理由不充分，脱离实际

### P8+ 级别（首席/领域负责人）

#### 题目类型1：跨域架构设计
**场景引入**：从简历中的复杂系统切入
**主问题**：[系统]涉及多个技术域，整体架构是如何设计的？

**追问层级**：
- L1（域间协作）：各个技术域如何分工协作？边界如何划分？
- L2（治理策略**：跨域一致性、性能、稳定性如何保障？
- L3（演进规划）：未来3年架构如何演进？如何支撑业务增长？

**参考答案要点**：
1. 能驾驭复杂系统的全局架构
2. 理解域间协作和治理策略
3. 有长期架构规划能力
4. 能平衡技术理想与业务现实

**评分标准**：
- 优秀：架构视野广，治理策略成熟，有战略思考
- 良好：架构完整，能描述协作机制，有规划意识
- 待观察：视野局限，缺乏治理思考，短视

#### 题目类型2：技术战略与业务赋能
**场景引入**：从简历中的技术影响力切入
**主问题**：你如何理解技术在业务发展中的作用？

**追问层级**：
- L1（价值认知）：技术如何创造业务价值？能举个例子吗？
- L2（战略思考）：技术战略如何与业务战略对齐？
- L3（组织影响）：如何推动技术战略在组织内落地？

**参考答案要点**：
1. 深刻理解技术商业化的逻辑
2. 能将技术战略与业务战略结合
3. 有组织影响力和推动能力
4. 有成功案例和复盘经验

**评分标准**：
- 优秀：认知深刻，案例具体，有战略高度
- 良好：理解正确，有实际案例，有一定高度
- 待观察：理解表面，脱离业务，缺乏案例

#### 题目类型3：行业趋势与技术判断
**场景引入**：开放性问题
**主问题**：你如何看待[某技术领域]的发展趋势？

**追问层级**：
- L1（现状理解）：当前该技术领域的主要玩家和方案是什么？
- L2（趋势判断**：未来3-5年会如何发展？为什么？
- L3（落地思考）：如果引入公司，你会怎么规划和推进？

**参考答案要点**：
1. 对技术领域有深入研究和洞察
2. 能独立判断技术趋势
3. 有技术预研和引入经验
4. 能制定可落地的技术规划

**评分标准**：
- 优秀：洞察深刻，判断有理有据，规划清晰
- 良好：有研究，能表达观点，有规划意识
- 待观察：了解表面，判断缺乏依据，无规划能力

## 第2轮：主管面试

> 调整重点：保留项目复盘与管理协作主线，减少底层原理深挖，强化 ownership、跨团队推进、冲突处理、优先级与压力下决策。

### 通用题目（适用于各级别）

#### 题目类型1：项目复盘与协作
**场景引入**：从简历中的重点项目切入
**主问题**：请介绍一个你最有成就感的项目

**追问层级**：
- L1（事实确认）：项目背景是什么？你的角色和贡献？
- L2（协作挑战）：项目中遇到过什么协作难题？如何解决的？
- L3（成长收获）：从这个项目中学到了什么？如果重来会怎么做？

**参考答案要点**：
1. 能清晰描述项目价值和自身贡献
2. 展示跨团队协作能力
3. 有反思和成长意识
4. 体现ownership和驱动力

**评分标准**：
- 优秀：故事完整，协作主动，成长深刻
- 良好：描述清晰，能解决协作问题，有收获
- 待观察：描述散乱，协作被动，缺乏反思

#### 题目类型2：冲突处理与沟通
**场景引入**：假设场景
**主问题**：如果你和产品经理在技术方案上有分歧，会怎么处理？

**追问层级**：
- L1（处理思路）：你的处理步骤是什么？
- L2（具体案例）：能举一个实际例子吗？
- L3（方法论）：如何避免或降低这类冲突？

**参考答案要点**：
1. 有成熟的冲突处理思路
2. 能平衡技术立场和业务诉求
3. 有实际案例支撑
4. 有预防和改进意识

**评分标准**：
- 优秀：思路成熟，案例具体，有方法论沉淀
- 良好：有思路，能举例，有一定方法
- 待观察：思路简单，缺乏案例，无方法

#### 题目类型3：时间管理与优先级
**场景引入**：工作场景
**主问题**：当你同时面对多个紧急任务时，如何安排优先级？

**追问层级**：
- L1（判断标准）：你的优先级判断标准是什么？
- L2（具体实践）：能举一个实际处理的例子吗？
- L3（压力应对）：如果领导要求的优先级和你判断的不一致怎么办？

**参考答案要点**：
1. 有清晰的优先级判断框架
2. 能结合业务价值和技术风险判断
3. 有实际执行经验
4. 能处理上级压力

**评分标准**：
- 优秀：框架清晰，案例丰富，能平衡多方诉求
- 良好：有判断标准，能举例，能沟通
- 待观察：标准模糊，缺乏案例，被动执行

## 第3轮：总监面试

> 调整重点：保留架构与战略题型，但更强调业务目标对齐、资源取舍、组织影响力和候选人的全局判断力，减少脱离业务的空泛趋势讨论。

### P7-P8 级别

#### 题目类型1：架构演进与治理
**场景引入**：从简历中的长期项目切入
**主问题**：[系统]的架构是如何随业务演进的？

**追问层级**：
- L1（演进历程）：架构经历过哪些重大变化？每次变化的原因是什么？
- L2（治理策略）：如何保障架构演进过程中的稳定性？
- L3（未来规划）：未来3年技术债务如何偿还？架构如何升级？

**参考答案要点**：
1. 能清晰描述架构演进历程
2. 理解技术债务的产生和治理
3. 有稳定性保障经验
4. 有长期技术规划能力

**评分标准**：
- 优秀：演进清晰，治理成熟，规划长远
- 良好：能描述演进，有治理意识，有规划
- 待观察：演进混乱，缺乏治理，短视

#### 题目类型2：技术影响力建设
**场景引入**：团队场景
**主问题**：你如何提升团队的技术能力？

**追问层级**：
- L1（具体措施）：做过哪些具体的事情？效果如何？
- L2（文化建设）：如何营造技术氛围？
- L3（长效机制**：如何保证技术影响力持续输出？

**参考答案要点**：
1. 有具体的技术分享和传承经验
2. 能营造团队技术氛围
3. 有体系化的能力建设思路
4. 有长期规划和执行能力

**评分标准**：
- 优秀：措施具体有效，文化营造成功，机制可持续
- 良好：有措施，有效果，有一定思路
- 待观察：措施零散，效果不明显，无长期思考

#### 题目类型3：业务理解与技术赋能
**场景引入**：业务场景
**主问题**：你如何理解技术在业务增长中的作用？

**追问层级**：
- L1（价值认知）：技术如何直接创造业务价值？
- L2（案例支撑）：举一个技术驱动业务的例子
- L3（战略思考）：技术如何支撑未来3年业务发展？

**参考答案要点**：
1. 深刻理解技术商业化的逻辑
2. 有技术驱动业务的成功案例
3. 能将技术与业务战略结合
4. 有前瞻性思考

**评分标准**：
- 优秀：理解深刻，案例有力，思考前瞻
- 良好：理解正确，有案例，有一定思考
- 待观察：理解表面，案例薄弱，缺乏战略

### P8+ 级别

#### 题目类型1：领域架构与战略
**场景引入**：开放场景
**主问题**：你如何看待[所属领域]的技术发展趋势？

**追问层级**：
- L1（领域现状）：当前领域的主要技术挑战是什么？

**参考答案要点**：
1. 对技术领域有深入研究和洞察
2. 能制定长期技术战略
3. 有组织影响力
4. 能平衡理想与现实

**评分标准**：
- 优秀：洞察深刻，战略清晰，影响力强
- 良好：有研究，有战略，有一定影响
- 待观察：了解表面，战略模糊，缺乏影响

#### 题目类型2：技术创新与探索
**场景引入**：创新场景
**主问题**：你在技术创新方面有什么探索？

**追问层级**：

**评分标准**：

#### 题目类型3：跨部门协同与影响
**场景引入**：协同场景
**主问题**：你如何推动跨部门的技术项目？

**追问层级**：

- L3（机制优化）：如何提升跨部门协同效率？

**参考答案要点**：
1. 有成功的跨部门项目经验
2. 能解决复杂的人际关系问题
3. 有方法论沉淀
4. 能建立协同机制

**评分标准**：
- 优秀：经验丰富，解决有力，机制完善
- 良好：有经验，能解决，有一定方法
- 待观察：经验薄弱，解决乏力，无方法

## 第4轮：HR面试

> 调整重点：保留动机与文化匹配主线，强化稳定性风险、自我认知、反馈接受度与预期管理，避免继续把技术深挖作为主轴。

### 通用题目

#### 题目类型1：离职动机与求职意愿
**场景引入**：开场
**主问题**：为什么考虑离开现在的公司？

**追问层级**：
 - L1（表面原因）：主要原因是什么？
- L2（深层驱动）：这个因素对你有多重要？
- L3（匹配验证）：你觉得我们公司能满足你的诉求吗？

**参考答案要点**：
1. 离职原因合理且真实
2. 有清晰的职业诉求
3. 对公司有了解和期待
4. 价值观与本公司匹配

**评分标准**：
- 优秀：原因真实合理，诉求清晰，匹配度高
- 良好：原因合理，有一定诉求，基本匹配
- 待观察：原因模糊或不当，诉求不清，匹配度低

#### 题目类型2：价值观与文化适配
**场景引入**：文化场景
**主问题**：你理想的工作环境是什么样的？

**追问层级**：
- L1（环境偏好）：你喜欢什么样的团队氛围和工作方式？
- L2（文化认同**：了解我们公司文化吗？哪些点吸引你？
- L3（适配评估）：如果文化有冲突，你会怎么处理？

**参考答案要点**：
1. 有清晰的工作环境偏好
2. 对公司文化有了解和认同
3. 能处理文化差异
4. 有融入团队的意愿

**评分标准**：
- 优秀：偏好清晰，认同度高，能主动融入
- 良好：有偏好，基本认同，能处理差异
- 待观察：偏好模糊，认同度低，融入困难

#### 题目类型3：职业规划与薪酬期望
**场景引入**：规划场景
**主问题**：未来3年的职业规划是什么？

**追问层级**：
- L1（规划描述）：具体想在哪方面发展？达到什么目标？
- L2（实现路径**：打算如何实现这些目标？
- L3（薪酬期望）：对薪酬有什么期望？依据是什么？

**参考答案要点**：
1. 有清晰的短期职业规划
2. 规划与公司发展匹配
3. 有可行的实现路径
4. 薪酬期望合理

**评分标准**：
- 优秀：规划清晰匹配，路径可行，期望合理
- 良好：有规划，基本匹配，期望适中
- 待观察：规划模糊或不匹配，期望过高

## 使用说明

### 追问策略

1. **保底3道追问**：每道题至少有3个层次的追问
2. **动态选择**：根据候选人回答质量选择性追问
   - 回答浅显 → 追问L1确保基础
   - 回答中等 → 追问L2挖掘深度
   - 回答深入 → 追问L3验证视野
3. **追问时机**：主问题回答完毕后，根据情况选择1-2个追问深入

### 时间管理

| 轮次 | 总时长 | 题目数 | 每题时长 |
|------|--------|--------|----------|
| 第1轮 | 30-40分钟 | 3题 | 10-15分钟/题 |
| 第2轮 | 30-40分钟 | 3题 | 10-15分钟/题 |
| 第3轮 | 30-40分钟 | 3题 | 10-15分钟/题 |
| 第4轮 | 20-30分钟 | 3题 | 7-10分钟/题 |

### 评分汇总

面试结束后，汇总每轮评分：
- 优秀：4-5分
- 良好：3分
- 待观察：1-2分

建议录用标准：
- 第1轮（技术专家）：≥3分（技术真实性与技术判断过关）
- 第2轮（主管）：≥3分（团队协作OK）
- 第3轮（总监）：≥3分（架构视野OK）
- 第4轮（HR）：≥3分（文化适配OK）

四轮均≥3分建议录用，有任一轮<3分需慎重考虑。
