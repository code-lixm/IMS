# Interview Questioning

## 输入契约

### 必填

- `round`：面试轮次（1-4）
- `level`：目标职级编码（仅允许 `P5-|P5|P5+|P6-|P6|P6+|P7-|P7|P7+|P8-|P8|P8+`）

### 来源优先级（强制）

1. 本地已存在轮次题目（`01_面试题_第{round}轮.md`，存在则直接复用）
2. `00_筛选报告.md`
3. 上一轮评价（`*.md` 或粘贴文本）
4. `resume.pdf`
5. 用户直接输入的岗位/JD 文案

### 可选

- `focus`：重点考察方向
- `previous_feedback`：上轮反馈（可直接粘贴）

### 轮次默认角色映射（强制）

- `round=1` → 技术专家面试
- `round=2` → 主管面试
- `round=3` → 总监面试
- `round=4` → HR 面试

## 输出契约

本 Skill 输出**可直接面试的 Markdown 文档内容**（不是纯 JSON 对象），由 `interview-records` 统一落盘为 `01_面试题_第{round}轮.md`。

## 生成规则（强制）

1. 每轮默认 3-5 题。
2. 每题必须包含：场景背景、主问题、L1-L3追问、追问策略、参考答案要点、评分标准。
3. 禁止再次询问“是否需要答案”；答案与评分标准必须默认同文输出。
4. 若输入已有筛选结论，题目必须覆盖筛选风险点。
5. 若输入有上一轮评价，题目必须覆盖上轮薄弱项并标注“延续自第N轮反馈”。
6. JD 缺失时允许按岗位基线出题，但必须在“候选人分析摘要”中标注 `JD缺失影响`。
7. 主问题禁止宽泛表达（如“聊聊你的项目经验/你怎么看前端发展”）；必须落到具体场景、约束条件和可验收结果。
8. 每题至少包含一个量化锚点（如耗时、并发、成功率、故障率、资源占用、SLA、成本）。
9. 题干语言必须口语化、面试现场可直接开口问；禁止模板腔（例如“请系统性阐述”“从多个维度分析”）。
10. 职级口径必须与 `interview-assessment` 一致：仅 `P5~P8` 三档制（`-/标准/+`），禁止 `P9+`、`Senior`、`P6.5` 等越界或别名定义。
11. 职级字段只允许输出级别编码（如 `P6`、`P7-`、`P8+`），禁止附加岗位称谓括注（如 `P6（高级前端工程师）`）。
12. 第1轮（技术专家）题目权重应偏向技术真实性、项目细节、方案取舍、问题排查与性能稳定性，技术类题目应占主轴。
13. 第2轮（主管）题目权重应偏向协作推进、ownership、冲突处理、优先级判断，可保留少量技术判断题，但不再以技术细节为主。
14. 第3轮（总监）题目权重应偏向架构视野、业务对齐、资源取舍、全局判断与组织影响力，减少执行层细节盘问。
15. 第4轮（HR）题目权重应偏向职业动机、稳定性、沟通成熟度、自我认知与文化匹配，避免继续深挖技术细节。
16. 第1-3轮都要保持场景化、证据导向的提问；第4轮语言可以更自然对话，但仍需保留可验证的行为追问。

## 年限分层出题策略（强制）

1. **短年限（0-3年）**：在项目贴合题之外，必须增加“技术视野 + 学习能力”题型（新技术调研、未知问题学习路径、踩坑复盘与迁移能力）。
2. **中年限（4-7年）**：在项目贴合题之外，必须增加“技术攻坚”题型（复杂故障定位、性能瓶颈治理、跨模块改造推进、风险兜底）。
3. **高年限（8年及以上）**：在项目贴合题之外，必须增加“管理策略/方法论/SOP”题型（团队协作机制、流程标准化、质量体系、成本与效率平衡）。
4. 题集结构必须同时包含：`项目实战题 + 年限分层题`，禁止只围绕项目场景单线出题。
5. 若年限信息缺失，默认按“中年限”策略生成，并在文档中标注“年限缺失影响”。

## 去 AI 味道约束（强制）

1. 优先使用候选人简历中的真实名词：项目名、技术栈、指标、业务对象。
2. 每题都要体现“为什么问这题”，并和候选人经历一一对应。
3. 禁止连续使用抽象词堆砌（如“架构能力/综合素养/全局视角”）而不给具体行为证据。
4. 题目正文默认不出现“AI生成/模型建议/自动化评估”等字样。
5. 输出前做一次“现场可读”自检：读出来像面试官提问，而不是方案文档。

## 与 records 的边界

- ✅ 本 Skill 负责“可直接面试的 Markdown 内容”。
- ❌ 本 Skill 不负责“阶段主文档命名与落盘”。

## 与 orchestrator 协作

`interview-orchestrator` 会：

1. 解析轮次并先检查本地同轮次题目是否可复用
2. 仅在缺失时调用本 Skill 生成
3. 使用 `interview-records.write_stage_document(stage=S1, round=N)` 落盘为 `01_面试题_第N轮.md`
