## 快速导航

### 🔧 基础能力 (Phase 1)
- **消息持久化**: SQLite 表设计、REST API、Pinia Store、组件集成
- **UI Bug 修复**: 重复会话、思考阶段展开/收起、候选人选择
- **Agent 简化**: 移除复杂说明，保留核心信息

### 🤖 Agent 增强 (Phase 2)
- **文件工具**: IndexedDB 存储、read/write/list 工具、资源列表组件
- **记忆系统**: 全局记忆、会话记忆、检索注入、自动提取

### 🏗️ 架构重构 (Phase 3)
- **LangGraph 架构**: Supervisor + Worker 模式
- **专业 Agent**: Screening / Question / Assessment / Email
- **工作流**: 状态管理、条件路由、并行执行

### 💼 业务功能 (Phase 4)
- **邮件系统**: SMTP 配置、模板管理、一键发送、发送记录
- **面试评估**: 多维度评分、自动计算、报告生成

### ✨ 体验优化 (Phase 5-6)
- **键盘监听**: Tauri 全局快捷键、浮动窗口、录音控制
- **引导页面**: 端点配置、登录、快速设置
- **问题修复**: 候选人显示、PDF 资源

## 实施建议

### 执行顺序

1. **立即开始**: Phase 1（基础能力，阻塞后续开发）
2. **并行开发**: 
   - Phase 2（文件/记忆）与 Phase 3（多 Agent）部分可并行
   - Phase 4（邮件/评估）可与 Phase 3 后期并行
   - Phase 5-6（优化）可与 Phase 3-4 并行
3. **里程碑检查点**:
   - Week 2: Phase 1 完成，基础功能可用
   - Week 5: Phase 2-3 完成，新 Agent 架构运行
   - Week 8: Phase 4 完成，业务闭环
   - Week 9: Phase 5-6 完成，全量优化上线

### 风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| LangGraph 学习成本 | Phase 3 延期 | 提前调研，预留缓冲时间 |
| Tauri 全局快捷键 | Phase 5 复杂 | 分阶段实现，先窗口后快捷键 |
| 邮件 SMTP 配置 | Phase 4 用户体验 | 提供常见邮箱预设配置 |
| 记忆系统效果 | Phase 2 价值 | MVP 版本，后续迭代优化 |

## 技术栈汇总

| 领域 | 技术/库 | 用途 |
|------|---------|------|
| 状态管理 | Pinia + IndexedDB | 前端状态 + 本地存储 |

| 数据库 | SQLite | 持久化存储 |
| AI 框架 | LangGraph + Vercel AI SDK | 多 Agent 工作流 |
| 邮件 | nodemailer | SMTP 发送 |
| 桌面 | Tauri v2 | 全局快捷键 + 浮动窗口 |
| UI | Vue 3 + shadcn-vue | 组件库 |
